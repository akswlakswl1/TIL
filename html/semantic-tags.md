# Semantic HTML

**Date:** 2025-05-04

## What I learned

HTML 태그는 의미(semantic)를 가져야 한다.
div 대신 목적에 맞는 태그를 써야 검색엔진과
스크린 리더가 구조를 이해할 수 있다.

## Key tags

| Tag       | Role                        |
|-----------|-----------------------------|
| header    | 페이지 상단 영역             |
| nav       | 네비게이션 링크 모음         |
| main      | 핵심 콘텐츠                  |
| section   | 관련 콘텐츠 묶음             |
| footer    | 하단 영역                    |

## head vs header

- head : 브라우저 설정 (화면에 안 보임)
- header : 페이지 상단 UI (화면에 보임)

## Block vs Inline

- block : div, p, h1 → 한 줄 전체 차지
- inline : span, a → 내용만큼만 차지
